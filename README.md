  ## 📝 Latest Updates

  ### Search Feature Update (Frontend)

  **Summary**
  - Integrate `wardrobeApi.search` and `wardrobeZoneApi.search` API calls in `wardrobeService.ts`.
  - Refactor `Topbar.tsx` search input to use local state (`localSearch`) combined with a `400ms` debounce to prevent React Router state interruptions and properly support Vietnamese IME composition (UniKey).
  - Integrate `useSearchParams` into `WardrobeManagement.tsx` and `WardrobeZones.tsx` to automatically trigger debounced data fetching upon URL query changes.
  - Gracefully handle `404 Not Found` responses generated by Backend's `AppException` by rendering an empty list state.

  **Search UI Flow**
  | Step | Actor | Action | Result |
  | :--- | :--- | :--- | :--- |
  | 1 | User | Type Vietnamese text into the Topbar Search Input | `localSearch` state updates smoothly without interrupting IME |
  | 2 | Topbar | User stops typing for 400ms (Debounce triggers) | Updates React Router's `searchParams` (`?q=keyword`) |
  | 3 | Router | URL search param updates | Components (`WardrobeManagement`, `WardrobeZones`) detect the parameter change |
  | 4 | Page | Call `wardrobeApi.search(q)` after a brief delay | API request is dispatched to BE |
  | 5 | Page | Intercept `404 Not Found` response containing `AppException` | Handle error gracefully, setting the list state to `[]` |
  | 6 | UI | Render UI with empty items | "Chưa có tủ đồ nào" / "Không tìm thấy vật phẩm" displays |
